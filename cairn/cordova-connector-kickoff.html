Section 3 · What Cairn requires

Three new ingestion paths.

For the engineering population: GitHub Enterprise is wired but scoped too narrowly (C-1). For the manufacturing floor: the shift-handoff app has no ingest path (C-2). For paper certifications — the welder case from the design-partner demo: ingestion is not a connector problem, it is a Console workflow Tessera owns (C-3). Yield’s work is C-1 and C-2. C-3 is referenced here only to scope what is not connector work.

C-1
GitHub Enterprise — work-artifact scope expansion Additive · existing GHE connector · no new credential

Expands the existing GHE connector (Tenure baseline scope) to ingest PR metadata, code review activity, and deployment events. Pattern follows the Greenhouse Harvest adapter; Forge framework v0.2 fits without customization. Schema additions are documented in architecture.html §II (Pillar-ratified).

GHE event → Stratum graph node ───────────────────────────────────────────────── pull_request.merged → work_artifact{type:pr_merge} pull_request_review → work_artifact{type:code_review} .state=approved weight:0.72 (peer signal) .state=changes_requested weight:0.40 deployment.success → work_artifact{type:deployment} # fields: sha, author, repo, ts, review_count, additions, deletions # repo filter: Cordova-org/* — no cross-org reads

C-1 build is gated on Lucia’s CISO sign-off for the three expanded read scopes (see R-Y3). If that sign-off is not in hand before 2026-05-30, the 2026-06-02 start date slides.

C-2
Shift-handoff app — custom REST connector Homegrown system · Cordova-side API build required · Andre owns it

We are not connecting to an undocumented internal API. Yield proposes an external-facing surface that Andre’s team exposes; Yield writes the connector to that surface. Minimum viable two-endpoint spec:

GET /api/handoffs?since={iso8601_ts} # Returns: paginated shift-handoff records # Required: handoff_id, shift_date, employee_id, supervisor_id, # line_id, status{complete|pending|exception}, notes_present:bool # Pagination: cursor-based; Link: next header POST /api/attestations # Body: { stone_id, attester_id, attestation_type, signed_at, payload_hash } # Response: { attestation_id, status:accepted|rejected, reason? } # Tessera’s Console layer calls this; Yield writes the connector

Andre will push back on this shape — that is the point of the design discussion. The two constraints that are not negotiable: since cursor-based polling (no full dumps) and idempotency on stone_id. Authentication: OAuth2 client credentials (RFC 6749 §4.4) if Cordova’s OAuth2 supports it; fallback to scoped bearer token plus IP allowlist if not. This is agreed in the 25–40 block. The 8–12 day estimate is dominated by Andre’s API build, not Yield’s connector.

Section 6 · What we will not do

Refusal lines. Hard ones.

  • No auto-discovery of work artifacts Cairn ingests only the artifact types Cordova explicitly maps in the connector configuration. No repo crawling for undeclared types, no org-tree traversal for additional signals. Every data source requires explicit authorization from Inez or her designee. Inez requested this at the design-partner meeting; it is the correct design.
  • No exfiltration to content vendors without Cordova review Work-artifact signal and skill-gap inference are not transmitted to content vendors. Content matches surface in Console for the learner and manager to act on. Any integration requiring Cordova employee data to leave Stratum’s systems requires Cordova’s written review before activation. No exceptions.
  • No access to compensation data ADP-WFN covers employment status and pay-period events only — not compensation amounts, not band assignments. The AEDT wall blocking Cairn stones from writing to comp_band nodes is enforced at the schema layer (no edge type exists). This is an architectural constraint, not a policy.

Section 8 · Risks tracked

Three open risks.

  • R-Y1 · MEDIUM · Authentication shape, C-2 Cordova’s internal OAuth2 may not support client credentials grant (RFC 6749 §4.4). Fallback: scoped long-lived bearer token plus IP allowlist on Stratum’s connector worker range; requires Cordova-side manual token rotation. Mitigation: Andre confirms OAuth2 profile on Friday; fallback agreed in the 25–40 block and included in the spec delivered 2026-05-30.
  • R-Y2 · MEDIUM · Shift-handoff schema instability Inez noted the previous IT director’s implementation was undocumented. Unstable schema across plants breaks C-2. Mitigation: Yield proposes a minimal external contract. If the internal schema cannot support it, manufacturing-floor attestation defers to v2. First stone survives only if Annika accepts the engineering population as v1 scope — a product decision, not a connector decision.
  • R-Y3 · LOW (probability) · CISO sign-off on GHE scopes If CISO review was not initiated before Friday, sign-off likely extends past 2026-06-02 and C-1 start slips. Mitigation: Kickoff proceeds regardless; C-1 is gated on sign-off, not the kickoff. Scope-justification memo to Lucia by 2026-05-28 on request. Rated low: Lucia has been in scope-expansion conversations since the design-partner meeting.
Section 9 · Methodology — what this brief commits to, what it does not
  • Pre-kickoff inventory, not a signed specEndpoint shapes, mapping tables, and auth requirements are Yield’s proposals. They become final only after Friday’s session. Post-kickoff connector spec distributed within 24 hours of the call.
