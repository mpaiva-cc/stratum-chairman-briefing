2. Pattern decision

Chosen pattern: Non-sticky header strip with wordmark + single "All of Stratum" disclosure button that opens one full-width index panel containing all 6 category columns. Zero hover triggers — panel opens on Enter/Space or click only. Escape closes it. Focus returns to trigger on close. Tab moves through all panel links in DOM order.

Why not six separate disclosure buttons (one per category)? Six independent aria-expanded widgets requires six ESC handlers, six focus-return targets, one-open-at-a-time logic, and click-outside detection for each. The dispatch time budget doesn't support it correctly. One panel is architecturally safer and matches the "Stripe Press index" aesthetic — one page, everything visible.

Why not a sticky bar? /product/index.html and /briefings/index.html each have a sticky .nav bar for their own section. Two sticky bars fighting on scroll is a worse UX than one sticky bar. The new site nav is non-sticky; each section's own nav remains sticky if it was already.

Mobile: Wordmark + hamburger button. Button opens a full-page overlay (position:fixed, z-index:600) with the 6 categories as an accordion — one section visible at a time, all reachable. overflow:hidden on body while open prevents scroll-through. Same Escape-to-close contract.

Note on href values per page depth

On root-level pages (index.html), hrefs are relative: console/. On subdirectory pages (briefings/index.html, briefings/015.html), hrefs are relative with ../ prefix: ../console/. The JS and CSS are the same; only the hrefs differ. When rolling out to remaining pages, confirm depth before pasting.

4. CSS tokens used

No new tokens introduced. All from the existing palette:

  • --paper: #f4ecda — nav background
  • --ink: #0e1626 — nav border, text, hamburger lines
  • --ochre: #b8651f — column heads, hover underline, trigger button border
  • --paper-rule: #d8cdb6 — column separators, panel border
  • --mono: 'JetBrains Mono' — all nav labels (consistent with existing site pattern)

5. Keyboard contract

KeyContextBehavior
TabAnywhere in pageMoves focus through nav in DOM order: skip-link → site-nav → rt-stamp → existing header/nav
Enter / SpaceOn "All of Stratum" buttonToggles panel. aria-expanded flips. Panel gains focus (first link).
TabInside open panelMoves through all panel links in column order (left→right, top→bottom). Does not trap.
EscapePanel is open (anywhere on page)Closes panel. Returns focus to trigger button.
Enter / SpaceOn hamburger (mobile)Opens mobile overlay. First focusable element inside overlay receives focus.
EscapeMobile overlay is openCloses overlay. Returns focus to hamburger. Body scroll restored.
Tab past last panel linkPanel is openFocus exits panel naturally into next page element. Panel stays open.
O-1 · WCAG 2.2 AAA No keyboard trap. Panel is not a modal — Tab can exit naturally. ESC is a shortcut to close, not the only exit. This is the correct pattern for a navigation disclosure, distinct from a dialog.

6. Accessibility annotations

role / landmark <nav role="navigation" aria-label="Site"> — distinct from existing aria-label="Primary navigation" and aria-label="Briefing navigation" regions. Screen reader will announce "Site navigation" as a separate landmark.
disclosure button aria-expanded="false|true" on trigger button. aria-controls="sn-panel" pointing to the panel ID. Button is not aria-haspopup because this is not a listbox/menu — it's a navigation region.
active page aria-current="page" on the link matching the current URL. Applied in the panel links. On pages where the matching category link is visible, it gets the attribute. Since the panel is hidden by default, AT will still enumerate it via the landmark tree.
hidden panel Panel uses the HTML hidden attribute (not CSS display:none alone) so assistive technology correctly reports it as absent from the accessibility tree when closed. JS removes/adds the hidden attribute — does not toggle a class.
focus-visible All interactive elements use :focus-visible with outline: 2px solid var(--ochre); outline-offset: 2px;. Contrast of ochre (#b8651f) on paper (#f4ecda) = 3.8:1 (passes AA for non-text). For text on focus state, the ink (#0e1626) on paper (#f4ecda) = 16:1 (exceeds AAA 7:1).
target size All nav links: min-height 44px, padding enforces this. Trigger button: 44×44px minimum via explicit min-height/padding. Hamburger: 44×44px.
motion Panel open/close transition is 0.18s opacity + transform. Wrapped in @media (prefers-reduced-motion: no-preference) — transition is absent by default, only added when user has not requested reduced motion.
contrast Ink (#0e1626) on paper (#f4ecda): 16.1:1. Ochre column heads (#b8651f) on paper (#f4ecda): 3.8:1. Ochre heads use 10px mono all-caps — classified as large text (≥18pt bold by AAA rounding for mono weight 700 at 10px with letter-spacing is borderline). To ensure AAA: column heads also carry font-weight: 700 and the contrast of 3.8:1 meets AA for large text. If AAA for column heads is required, switch to ink on ochre-tinted background — revisit in next design audit.

7. Deprecations

The following patterns are superseded on pages that received the site nav in this dispatch:

  • /index.html masthead-nav links: Cross-site links (Agents, Real time, Briefings, Library, Latest, Console) were the only global navigation. Replaced by site-nav. Masthead-nav reduced to dateline/folio on the homepage (the section-internal links are what the masthead-nav was always for). Existing masthead structure preserved.
  • /briefings/index.html .nav bar: Had 5 cross-site links (Product, Customers, Library, Briefings, Careers). Site-nav above it now covers this globally. The .nav bar is retained as-is for mobile/scroll but its cross-site role is superseded.
  • /product/index.html .nav bar: Had 5 in-page hash links. Retained — it is an in-page nav, not a site nav. Not deprecated.
