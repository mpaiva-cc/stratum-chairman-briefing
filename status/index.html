§ 03 · Incident history

Every event, with a root cause.

We publish a record of every operational event, including those caused by upstream providers. We do not redact severity, duration, or remediation.

T+-14d · resolved P2 · Degraded

Degraded performance — agent inference

Detected14:08 PT
Resolved15:41 PT
Duration93 min
Customer impactLatency only
Summary

Anthropic's API experienced elevated latency (p99 from 1.8s to 6.4s). Stratum's agent responses were slow but not failed. No customer-data impact. No data loss. We surfaced a banner in the Console for the duration explaining the upstream cause.

Root cause

Upstream provider (Anthropic) capacity event. We received no advance notice. Anthropic's own status page acknowledged the event at 14:22 PT, fourteen minutes after our internal probes detected it.

Remediation
  • Circuit breaker implemented: switch from Opus to Sonnet when Opus p99 latency exceeds 4.0s for 5 minutes.
  • Graceful-degradation banner now ships by default — explains the situation to end users with an ETA when available.
  • Requested capacity visibility / advance-notice channel from Anthropic for the future. In progress.
Post-mortem

Full post-mortem document available under NDA on request. Email security@stratum.ai.

T+-32d · resolved P3 · Informational

Brief connector backlog — ADP Workforce Now

Detected09:32 PT
Resolved09:58 PT
Duration26 min
Customer impactNone
Summary

ADP-WFN connector experienced a 26-minute ingestion lag (records were 26 minutes stale at peak). No customer-facing impact: fallback poll backfilled records before any downstream consumer queried them.

Root cause

ADP's webhook delivery experienced an upstream queue. Caught and remediated within 1 minute of detection via our pre-existing fallback poll.

Remediation

We had already implemented a fallback poll for ADP's webhook delivery; this incident validated the fallback worked correctly. No further action required, though we tightened the poll cadence from 5 minutes to 2 minutes for ADP-WFN specifically.

No prior public-facing incidents. Stratum was founded T+0d; all operational events before the events above were pre-production and therefore not customer-impacting.
