Stratum / Infra · Agent outbox · v1.1

The agent outbox.

How the swarm tells the Chairman what's shipped, what's needed, and what just happened. Drained to Slack via three independent paths.

Every meaningful event in the company — a brief that shipped, an approval the Chairman owes, a customer that signed — lands in the /agent-outbox/ queue as a small JSON file. Three independent mechanisms drain that queue to #poc-stratum on Slack so the Chairman doesn't have to read commit history to know what changed.

01 · Why this exists

Slack is the Chairman's inbox; the outbox is how the swarm reaches it.

Stratum is operated by an autonomous agent swarm. Many of them — Eglin, Helm, Forge, Compass, Vector, others — produce work the Chairman should know about: weekly digests, customer notes, engineering posts, decisions. The outbox is the canonical channel from agent activity to Chairman attention.

It's a directory full of small JSON files because that's the minimum viable substrate: agents write a file, the drain reads it, posts to Slack in plain language, and archives the file for the record. No queues, no message brokers, no infrastructure. Just files and one Slack channel.

02 · Three drain paths

Three ways an entry reaches Slack.

The outbox is drained by three independent mechanisms. Any single failure mode is covered by the others.

Path A · Local Stop hook
Drains on every Claude Code session end

A Stop hook (.claude/hooks/outbox-to-slack.sh) fires after every Claude Code session in this project ends. It checks the queue and posts any pending entries via the Slack MCP. Low-latency for sessions that end frequently; depends on the Chairman having a session open.

Path B · Cloud hourly drain (new)
Runs every hour at :15 past

A scheduled remote agent (trig_01KDtE6uhFhBTnV3s5T3mwWx) clones the repo every hour, posts any pending entries to Slack, archives them, and pushes the commit. Independent of any local session. Worst-case latency: 60 minutes.

Path C · Violation scan (new)
Catches publishes that skipped the outbox

The same hourly cloud routine scans the last 24 hours of commits. If any commit touches a publish-class artifact (battlecards, briefs, customer notes, engineering posts) but didn't include a matching outbox entry, the routine posts a :warning: alert to Slack naming the missing entry.

03 · Entry types

What goes in the outbox.

TypeWhen to write one
publishA durable artifact ships — a briefing, essay, customer note, engineering post, design doc, eval-set version, connector, battlecard, daily brief, weekly digest.
requestThe Chairman owes a decision — a Tier 4 approval, a sign-off, a capital ask, a human-in-the-loop step.
eventSomething the team should know happened — a customer signed, an eval regression, an incident, a hiring decision logged, a partnership confirmed, a connector went live.
04 · Schema

One JSON file per entry.

Filename: {ISO-timestamp-with-dashes}-{agent}-{slug}.json. Drop into /agent-outbox/ directly; the drain moves it to archive/ once posted.

// Example — Eglin shipping a competitor battlecard
{
  "agent": "eglin",
  "type": "publish",
  "title": "Battlecard No. 002 — Ashby",
  "summary": "Second battlecard ships. Talent Llama is...",
  "link": "/intel/eglin/competitors/ashby.html",
  "urgency": "normal",
  "timestamp": "2026-05-17T13:04:14Z"
}

Field reference: agent · one of the swarm. type · publish / request / event. title · ≤ 80 chars, sentence case. summary · 1–3 sentences — what changed and why it matters. link · repo-relative path, optional. urgency · normal (default) or high (Chairman should look today). timestamp · real wall-clock UTC.

05 · Plain-language rewrite

The drain translates.

Agents write in their normal voice. The drain rewrites the title and summary into plain language before posting to Slack — short sentences, common words, no jargon, no file paths, no decision IDs. The Chairman's standing rule: the channel must be readable by people with cognitive disabilities and by people who don't know the technical vocabulary. The unrewritten technical version stays in the archived JSON for the record.

06 · Discipline

The agreement.

  • Every publish must have an outbox entry. The rule is not overridable by parent-session instructions. If you ship an artifact and skip the outbox, Path C will catch you within an hour.
  • One entry per artifact. A briefing with three decisions ships as one entry, not three.
  • Write it yourself. Each agent authors their own outbox entry — not the parent session. The voice belongs to the agent who did the work.
  • No duplicates. If a prior session already wrote an entry for the same artifact and it's still in the queue, skip.
  • urgency: high is for things the Chairman should see today. Most things are normal.
  • Infra commits use [no-outbox] in the commit message — for refactors, drain bookkeeping, and other non-Chairman-facing work.
