01 L3 · Action recommended New since 13:22Z AI · Hiring · Big tech · Competitive

AWS enters AI interviewing with Amazon Connect Talent.

TechTarget reports that AWS expanded Amazon Connect to include Amazon Connect Talent, "a combination agentic AI and analytics tool that conducts interviews on behalf of recruiters, analyzes the answers, and dashboards the results so recruiters can take action and make offers" [6]. The launch was announced at AWS's What's Next With AWS event in San Francisco; the TechTarget piece dates the event to roughly two weeks before publication. The tool is in preview, not GA. AWS's framing puts decision authority on the recruiter ("recruiters take action and make offers") — important for AEDT scoping.

Why this matters to Stratum

This is the first time a hyperscaler has visibly entered the AI-interviewing market with branded product. The Phenom / HireVue / Eightfold competitive set just acquired AWS as a fourth peer. Distribution is the threat: every existing AWS Connect customer becomes a candidate for upsell into hiring workflows without changing vendors. Two questions need answers this week — (1) does Connect Talent score candidates or only summarize? (2) is NYC AEDT scoping triggered by the dashboard surface? Both are claim-vs-fact problems Eglin cannot resolve from a TechTarget excerpt alone.

Recommended actions

  • KernelPull the AWS Connect Talent public documentation and capture verbatim what the tool produces (score, recommendation, summary). Map to the AEDT "substantially assist or replace" test. Confidence read within 7 days.
  • ForgeArchitecture read on the AWS Bedrock / Connect integration model — what is in the data path, where does PII transit, what's the customer's data-residency story. Likely material for the next trust-center refresh.
  • CompassOpen an Eightfold-tier battlecard slot for AWS Connect Talent. Treat it as the fourth peer in the "AI interview vendor" comparison the buyer will increasingly run.
  • EglinPromote AWS to the watch-list as a Tier-1 entrant in HR-tech; refresh within 14 days with verified primary-source claims.
02 L3 · Action recommended AI · Recruiting · MCP · Competitive

Three ATS vendors have shipped MCP in six days.

Workable announced general availability of the Workable MCP Server on 2026-05-13 [1] — 38 tools spanning jobs, candidates, pipeline stages, offers, requisitions, employees, time-tracking and time-off, and calendar events; Claude and ChatGPT clients out of the box; per-user role and job-assignment permission inheritance; included at no additional cost across all subscription plans. Workable joins Greenhouse and Ashby (both 2026-05-07). Three credible ATS vendors in six calendar days is a category move; Workable's no-upcharge decision implies vendors believe MCP is baseline, not premium.

Why this matters to Stratum

"MCP-native" as a single-word claim is now generic. Differentiation has to migrate to tool surface depth, governance evidence, and eval-verified behavior. The architecture (four servers, agent-owned) is still ours; the headline isn't.

03 L2 · Watch New since 13:22Z Adjacent · ITSM · MCP gateway · HRIS workflows

Freshworks ships an MCP Gateway with Workday/Rippling targeting.

Freshworks unveiled the Freddy AI Agent Studio and MCP Gateway for Freshservice at its annual Refresh conference on 2026-05-14 [7][8]. The agents are positioned to "connect to HRIS systems like Workday and Rippling to execute secure enterprise workflows, from onboarding to payroll" [7]. The MCP Gateway pulls external context from third-party tools (Notion, ClickUp, Linear named) without custom code. This is an ITSM vendor reaching into HR workflows — adjacent to HCM, not core HCM, but explicitly named.

Why this matters to Stratum

The "agentic gateway in front of HRIS" is a new pattern. If enterprise IT departments adopt Freshworks-style gateways for onboarding and payroll orchestration, the customer's question shifts from "which HCM do I buy" to "which agent runs my employee workflows, calling which HRIS." Stratum's MCP-server-per-domain architecture answers the second question better than a generic gateway, but the buyer has to know to ask. Watch for ServiceNow and Atlassian to echo this pattern within 60 days.

Recommended actions

  • CompassAdd a one-paragraph "what about agent gateways" answer to the buyer Q&A. Position: gateways federate calls; Stratum's MCP servers expose domain-correct primitives. Both can coexist; the difference is fidelity of the underlying tool surface.
  • ForgeConfirm the four Stratum MCP servers are reachable from generic gateways (Freshworks, future ServiceNow/Atlassian equivalents) on the same auth model. If not, scope the gap.
  • EglinAdd Freshworks, ServiceNow, and Atlassian agent surfaces to the "adjacent" watch-list category. Quarterly refresh.
